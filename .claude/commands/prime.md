# Prime - Understand Codebase

Understand the codebase structure and key components before making any changes.

## Instructions

1. **List all files in the project**
   ```bash
   git ls-files
   ```

2. **Read the project README**
   - Locate and read README.md or similar documentation
   - Understand the project's purpose, architecture, and key technologies

3. **Identify key directories and their purposes**
   - Source code directories
   - Configuration files
   - Test directories
   - Documentation

4. **Summarize your understanding**
   - Project type and purpose
   - Main technologies and frameworks
   - Key architectural patterns
   - Testing approach (if present)

## Report

Provide a concise summary (3-5 bullet points) of:
- What this project does
- Key technologies used
- Main code organization
- Any critical patterns or conventions you've identified
